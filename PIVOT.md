I've added a new folder: @template . I believe that this is a very efficient way of working nowadays with Cursor & AI. Now this is copied, we have to make our own version out of this that can collaborate heavily with Trail. Now let's think of how to do this in combination with Trail. How I see it is this:

- Trail is the 'UI' and the 'Tool' to set your repo up in such a way and to work according to that process. Trail's role in this is to make sure everything we do is synced with Github (in terms of task management) and to visualise our progress. Github Issues is great for management, but poor for visualising and planning. Reading JSON files on the other hand is also horrible for human reading. Trail is the bridge between AI efficiency (with local files) and a human that can navigate through it.
- Trail should show, per task, how it's picked up, if there are any comments, basically to 'read the mind' of the AI as if a colleague has given a status update with arbirtrary comments to keep in mind.
- When setting up Trail I believe the items in the template folder should be scaffolded and added to the repository. As Trail is a way of working, you choose to work like that.

Trail is not:
- A replacement of Cursor. You will always use the Cursor UI to chat and start a Task.
- A replacement of Github. Everything we do should be synced with Github Issues, if the user selects to go this route.
